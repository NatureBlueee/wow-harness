图 2三层 harness — 每层在解决什么问题

v3 · 自治层
解决：harness 自己跑出 bug 时谁来修 harness（修问题反而引入问题）
改 harness 时不许重现自己要修的症状
H0 元规约 6+1 条款 · 修过度 review 的 PR 不许过度 review · grep 反查 · Nature 单签
多窗口并行不靠人手当中转
H9 文件邮箱 · 子窗口写 markdown 消息 → 主窗口启动 hook 自动注入到 prompt · 5 个 hook 保证不丢/不重
同样的坑不踩第二次
H1 crystal-learn · 真实事故 → 提取成不变量 → 注入回执行层 skill · 8 条已落地
编号撞 / 记忆涨爆 / 状态文件漂移 / 文档脱节
H2-H8 · 身份段独立 · MEMORY 主索引 ≤200 行 · ADR-INDEX 自指过滤 · open issue ≤80 等硬指标
↑ 当 v1+v2 自己跑出问题
v2 · 阶段化层
解决：AI 不知道当前在哪个阶段，工具权限一直全开
阶段切换自动收/放权
/mode plan → build → verify → release · plan 阶段 AI 只读不写 / build 阶段才解锁 Edit / verify 阶段强制看 evidence
审查标准写在契约里不在提示词里漂
review-contract.yaml 锁死「谁审什么 / 要什么证据 / 什么标准算通过」· verify-gate hook 在 Stop 时校验 evidence packet
↑ 当 v1 还不够
v1 · 物理拦截层
解决：AI 写规则但不听规则（CLAUDE.md 遵从率仅 ~20%）
想做错事就在动作那一刻挡下
18 hooks · 想 rm -rf？想 push 到生产？想跳测试？想暗中改审查 AI？hook 直接 exit 2 物理阻断
把 AI 拆成各司其职的角色
16 skills · lead 锁问题不写代码 / arch 出方案不实现 / dev 实现不审查 / reviewer 物理无写权 — 单 AI 干所有事必出错
不靠 AI 记规则，靠脚本自动验
15 checks · 撞 ADR 编号 / 引用错文件路径 / hook IO 不合规 — git pre-commit 拒提交，不靠人记
